# 产品需求文档（PRD）
## 经济学考研AI助手系统

---

## 1. 项目概述

### 1.1 项目背景
经济学考研竞争激烈，考生需要系统学习微观经济学、宏观经济学、政治经济学、计量经济学等多个学科，同时需要了解目标院校的分数线、招生人数等信息。传统的备考方式缺乏个性化指导和智能化练习，考生难以高效掌握知识点和真题规律。

### 1.2 项目目标
- 为经济学考研考生提供AI驱动的智能学习平台
- 实现知识点系统学习、真题模拟冲刺、错题智能管理
- 提供院校分数线和招生数据查询及AI咨询服务
- 帮助考生提高备考效率，科学规划复习路径

### 1.3 预期成果
- 完整的Web应用系统，支持PC端访问
- 集成AI导师、练习模块、错题管理、数据查询四大核心功能
- 用户可根据目标院校和专业定制学习方案
- 提供智能题目生成、答案评估、知识点推荐等AI能力

---

## 2. 用户分析

### 2.1 目标用户群体
- **主要用户**：经济学考研备考生（专硕/学硕）
- **次要用户**：经济学专业本科生、在职考研人员

### 2.2 用户画像

**用户画像1：应届本科生**
- 年龄：21-23岁
- 特征：时间充裕，学习能力强，需要系统化备考方案
- 痛点：知识点繁多，不知从何入手；缺乏真题练习资源

**用户画像2：在职考研人员**
- 年龄：24-30岁
- 特征：时间有限，需要高效利用碎片时间
- 痛点：工作繁忙，难以系统复习；需要针对性练习薄弱知识点

**用户画像3：二战考生**
- 年龄：22-25岁
- 特征：有一定基础，需要查漏补缺和冲刺提分
- 痛点：需要错题管理和针对性强化；需要了解目标院校最新招生信息

### 2.3 用户需求分析
- **学习需求**：系统学习各科目知识点，掌握核心概念和理论
- **练习需求**：真题练习、模拟考试、错题重练
- **咨询需求**：AI导师答疑、院校信息查询、志愿填报建议
- **管理需求**：错题收藏、学习进度追踪、个性化推荐

### 3.2 功能清单与优先级

#### P0 - 核心功能（必须实现）

**F1. 用户目标设置**
- 功能描述：用户首次进入系统时设置考研目标（专硕/学硕、目标院校、目标专业）
- 验收标准：
  - 支持选择考试类型（专硕/学硕）
  - 提供主流院校列表（北大、清华、复旦等）
  - 提供经济学相关专业列表
  - 配置信息持久化到localStorage
  - 支持后续修改目标

**F2. AI导师对话**
- 功能描述：基于Gemini AI的智能对话系统，回答考研相关问题
- 验收标准：
  - 支持多轮对话，保持上下文
  - 能够回答知识点问题、院校信息、备考建议
  - 响应时间<5秒
  - 显示加载状态
  - 对话历史保存在会话中

**F3. 系统学习（章节练习）**
- 功能描述：按科目和知识点进行系统化练习
- 验收标准：
  - 支持5个科目：微观经济学、宏观经济学、政治经济学、计量经济学、数学三
  - 每个科目包含4-6个核心知识点
  - 优先使用真题库，无真题时AI生成模拟题
  - 支持单选题、问答题、计算题三种题型
  - 提交答案后AI评估正确性并给出详细解析
  - 答对后可继续下一题，答错后推荐继续练习该知识点
  - 显示知识点和推荐课程（B站链接）

**F4. 模拟冲刺**
- 功能描述：基于真题的模拟考试练习
- 验收标准：
  - 支持按知识点筛选真题
  - 支持按年份筛选真题（2020-2024）
  - 每次练习最多10题
  - 显示题目来源（如"南京审计大学2024年真题"）
  - 完成后显示正确率统计
  - 支持收藏题目到错题本

**F5. 错题集管理**
- 功能描述：收藏、管理和重练错题
- 验收标准：
  - 默认"所有错题"集合
  - 支持创建自定义错题本（如"微观经济学"、"宏观经济学"）
  - 支持将题目添加到多个错题本
  - 错题卡片显示科目、知识点、题目预览
  - 点击错题查看详情（题目、答案、解析）
  - 支持错题重练
  - 支持生成5道相似题型练习
  - 错题数据持久化到localStorage

**F6. 分数线与招生数据查询**
- 功能描述：查询院校历年分数线和招生人数，并提供AI咨询
- 验收标准：
  - 分数线表格：年份、院校、省份、类型、专业、总分、单科分数、自划线、专项计划
  - 招生数据表格：年份、院校、省份、类型、专业、招生人数、报考人数、报录比
  - 支持搜索和筛选
  - 支持表格扩大/缩小查看
  - 右侧AI聊天窗口，基于数据提供报考建议
  - AI能够分析数据并给出个性化建议

#### P1 - 重要功能（优先实现）

**F7. 练习进度统计**
- 功能描述：统计用户练习情况，显示正确率、完成题数
- 验收标准：
  - 练习完成后显示统计数据（正确数、错误数、正确率）
  - 支持"再练一次"功能

**F8. 知识点推荐课程**
- 功能描述：答错题目后推荐相关B站课程
- 验收标准：
  - AI生成推荐课程名称
  - 提供B站搜索链接
  - 点击跳转到B站搜索结果页

**F9. 题目收藏功能**
- 功能描述：在练习过程中收藏题目到错题本
- 验收标准：
  - 答题后显示"收藏到错题本"按钮
  - 弹窗选择目标错题本
  - 支持收藏到多个错题本
  - 显示收藏成功提示

#### P2 - 优化功能（后续迭代）

**F10. 学习计划制定**
- 功能描述：AI根据目标院校和当前水平制定学习计划

**F11. 学习数据分析**
- 功能描述：可视化展示学习进度、薄弱知识点、练习趋势

**F12. 社区功能**
- 功能描述：考研经验分享、问题讨论、资料共享

## 4. 非功能需求

### 4.1 性能要求
- **响应时间**：
  - 页面加载时间 < 2秒
  - AI对话响应时间 < 5秒
  - 题目生成时间 < 3秒
  - 答案评估时间 < 3秒
- **并发能力**：支持100+用户同时在线
- **数据加载**：表格数据支持虚拟滚动，优化大数据量渲染

### 4.2 安全要求
- **数据安全**：
  - 用户数据存储在本地localStorage，不上传服务器
  - API密钥安全管理，不暴露在前端代码
  - 防止XSS攻击和注入攻击
- **隐私保护**：
  - 不收集用户个人敏感信息
  - AI对话内容仅在会话期间保存

### 4.3 可用性要求
- **界面友好**：
  - 采用现代化UI设计，色彩搭配合理
  - 响应式布局，适配不同屏幕尺寸
  - 操作流程简洁，减少用户学习成本
- **交互体验**：
  - 加载状态明确提示
  - 操作反馈及时（如收藏成功提示）
  - 支持键盘快捷键（如Enter提交）
- **错误处理**：
  - API调用失败时显示友好错误提示
  - 网络异常时提供重试机制

### 4.4 兼容性要求
- **浏览器兼容**：支持Chrome、Edge、Firefox、Safari最新版本
- **设备兼容**：优先支持PC端，兼容平板设备
- **分辨率**：支持1920x1080、1366x768等主流分辨率

## 5. 系统架构

### 5.1 技术选型

**前端技术栈**
- **框架**：React 18 + TypeScript
- **UI库**：Tailwind CSS（原子化CSS）
- **动画库**：Framer Motion (motion/react)
- **图标库**：Lucide React
- **状态管理**：React Hooks (useState, useEffect)
- **数据持久化**：localStorage
- **构建工具**：Vite

**AI服务**
- **AI模型**：Google Gemini API
- **功能**：
  - 题目生成（generateQuestion）
  - 答案评估（evaluateAnswer）
  - 相似题生成（generateSimilarQuestions）
  - AI对话（chatWithTutor）

**数据存储**
- **本地存储**：localStorage
  - 用户配置（userProfile）
  - 错题数据（wrongQuestions）
  - 错题本分类（collections）
- **静态数据**：
  - 真题库（REAL_EXAM_QUESTIONS）
  - 分数线数据（MOCK_SCORES）
  - 招生数据（MOCK_ENROLLMENT）

### 5.3 数据流说明

**练习流程数据流**
1. 用户选择科目和知识点
2. 系统从真题库筛选题目，无匹配时调用Gemini API生成题目
3. 用户提交答案
4. 调用Gemini API评估答案，返回正确性、解析、推荐课程
5. 用户选择收藏到错题本，数据保存到localStorage

**AI对话数据流**
1. 用户输入问题
2. 系统收集对话历史、用户配置、院校数据
3. 调用Gemini API生成回复
4. 返回AI回复并显示

**错题管理数据流**
1. 用户收藏题目时，题目添加到wrongQuestions数组
2. 题目关联到一个或多个错题本（collectionIds）
3. 数据持久化到localStorage
4. 用户重练时，从localStorage读取题目
5. 生成相似题时，调用Gemini API基于原题生成5道新题

## 6. 接口需求

### 6.1 外部API依赖

**Gemini AI API**
- **接口地址**：Google Gemini API
- **认证方式**：API Key
- **调用场景**：
  - 题目生成
  - 答案评估
  - 相似题生成
  - AI对话
- **请求频率限制**：根据Gemini API配额
- **错误处理**：
  - 超时重试（最多3次）
  - 失败时显示友好提示
  - 降级方案：使用静态题库

### 6.2 内部接口规划

**geminiService.ts 接口**

```typescript
// 生成题目
generateQuestion(
  subject: string,        // 科目
  knowledgePoint: string, // 知识点
  type: 'choice' | 'qa' | 'calculation' // 题型
): Promise<Question>

// 评估答案
evaluateAnswer(
  question: Question,     // 题目对象
  userAnswer: string      // 用户答案
): Promise<{
  isCorrect: boolean,
  analysis: string,
  detailedExplanation: string,
  recommendedCourse?: string
}>

// 生成相似题目
generateSimilarQuestions(
  question: Question,     // 原题
  count: number          // 生成数量
): Promise<Question[]>

// AI对话
chatWithTutor(
  history: Message[],     // 对话历史
  userMessage: string,    // 用户消息
  userProfile: UserProfile, // 用户配置
  enrollmentData: EnrollmentData[], // 招生数据
  scoreData: ScoreLine[]  // 分数线数据
): Promise<string>
```

### 8.3 风险评估

**技术风险**
- **风险1：AI API调用失败**
  - 影响：题目生成、答案评估功能不可用
  - 应对：实现降级方案，使用静态题库；增加重试机制；显示友好错误提示

- **风险2：AI生成内容质量不稳定**
  - 影响：题目质量参差不齐，解析不够准确
  - 应对：优化Prompt设计；人工审核高频题目；建立题目质量反馈机制

- **风险3：localStorage容量限制**
  - 影响：错题数据过多时可能超出存储限制（通常5-10MB）
  - 应对：定期清理旧数据；实现数据压缩；引入IndexedDB

**业务风险**
- **风险4：真题版权问题**
  - 影响：可能涉及版权纠纷
  - 应对：仅用于学习研究；标注题目来源；必要时移除敏感内容

- **风险5：用户数据安全**
  - 影响：localStorage数据可能被恶意脚本读取
  - 应对：不存储敏感信息；提示用户不在公共设备使用；考虑数据加密

**运营风险**
- **风险6：AI成本过高**
  - 影响：大量用户使用时API调用费用激增
  - 应对：实现请求频率限制；缓存常见问题答案；考虑自建模型

## 9. 数据字典

### 9.1 核心数据结构

**UserProfile - 用户配置**
```typescript
{
  target: '专硕' | '学硕' | null,  // 考试类型
  university: string | null,       // 目标院校
  major: string | null             // 目标专业
}
```

**Question - 题目**
```typescript
{
  id: string,                      // 题目ID
  year?: number,                   // 年份（真题）
  source?: string,                 // 来源（如"南京审计大学2024年真题"）
  type: 'choice' | 'qa' | 'calculation', // 题型
  content: string,                 // 题目内容
  options?: string[],              // 选项（单选题）
  answer: string,                  // 正确答案
  analysis: string,                // 解析
  knowledgePoint: string,          // 知识点
  subject: string                  // 科目
}
```

**WrongQuestion - 错题**
```typescript
{
  ...Question,                     // 继承Question所有字段
  addedAt: number,                 // 添加时间戳
  collectionIds: string[]          // 所属错题本ID列表
}
```

**WrongCollection - 错题本**
```typescript
{
  id: string,                      // 错题本ID
  name: string                     // 错题本名称
}
```

**ScoreLine - 分数线**
```typescript
{
  id: string,
  year: number,                    // 年份
  university: string,              // 院校
  province: string,                // 省份
  type: '学硕' | '专硕',           // 类型
  major: string,                   // 专业
  totalScore: number,              // 总分
  singleSubject1: number,          // 单科1分数
  singleSubject2: number,          // 单科2分数
  selfDrawn: boolean,              // 是否自划线
  specialPlan: boolean             // 是否专项计划
}
```

**EnrollmentData - 招生数据**
```typescript
{
  id: string,
  year: number,                    // 年份
  university: string,              // 院校
  province: string,                // 省份
  type: '学硕' | '专硕',           // 类型
  major: string,                   // 专业
  enrollment: number,              // 招生人数
  applicants: number,              // 报考人数
  ratio: string                    // 报录比
}
```

### 9.2 localStorage存储键

- `userProfile`: 用户配置
- `wrongQuestions`: 错题数据数组
- `collections`: 错题本分类数组

## 10. UI/UX设计规范

### 10.1 色彩系统
- **主色调**：Indigo（靛蓝）- 专业、可信赖
  - Primary: `#4F46E5` (indigo-600)
  - Light: `#EEF2FF` (indigo-50)
- **辅助色**：
  - Purple（紫色）- 模拟冲刺模块
  - Blue（蓝色）- 系统学习模块
  - Rose（玫瑰红）- 错题集模块
  - Emerald（翠绿）- 成功状态
- **中性色**：
  - Slate系列 - 文本、边框、背景

### 10.2 组件规范
- **按钮**：圆角12px，悬停时加深颜色
- **卡片**：圆角16-24px，边框1px，阴影subtle
- **输入框**：圆角12px，focus时显示ring
- **标签**：圆角full，小号字体，大写
- **表格**：斑马纹，悬停高亮

### 10.3 交互规范
- **加载状态**：显示Loader2旋转图标 + 文字提示
- **成功反馈**：绿色背景 + Check图标 + 2秒自动消失
- **错误提示**：红色背景 + X图标 + 友好文案
- **动画**：使用Framer Motion，过渡时间200-300ms
